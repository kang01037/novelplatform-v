# 小说平台接口文档

## 1. 文档说明

本文档详细描述了小说平台的所有后端接口，包括用户、小说、章节、书架、评论和微信登录模块。所有接口均基于RESTful风格设计，使用JSON格式进行数据交互。

## 2. 响应格式

所有接口的响应格式统一为：

```json
{
  "code": 200,
  "message": "success",
  "data": "返回数据"
}
```

- `code`: 响应状态码（Integer），200表示成功，其他表示失败
- `message`: 响应消息（String），描述操作结果
- `data`: 响应数据，根据接口不同返回不同格式的数据

## 3. 认证说明

### 3.1 双 Token 机制

本项目使用双 JWT（Access Token + Refresh Token）认证机制：

| Token 类型 | 有效期 | 用途 | 存储位置 |
|-----------|--------|------|----------|
| Access Token | 30 分钟（可配置） | 请求认证，每次请求通过 `Authorization` 头携带 | 前端内存 / localStorage |
| Refresh Token | 7 天（可配置） | 获取新的 Access Token，同时存储在 Redis 中用于吊销 | localStorage + Redis |

### 3.2 认证流程

1. 用户登录成功后，服务端返回 `accessToken`（短期）和 `refreshToken`（长期）
2. 前端所有需要认证的请求，在 `Authorization` 头中携带 `Bearer {accessToken}`
3. Access Token 过期后，前端自动使用 `refreshToken` 调用 `/api/auth/refresh` 获取新的 token 对
4. Refresh Token 也过期或无效时，用户需要重新登录

### 3.3 认证请求头

需要认证的接口需要在请求头中添加：

```
Authorization: Bearer {accessToken}
```

## 10. 认证模块

### 10.1 接口列表

| 接口名 | URL | 方法 | 功能描述 | 认证要求 |
|-------|-----|------|----------|----------|
| 刷新 Token | /api/auth/refresh | POST | 使用 refreshToken 获取新的 token 对 | 无需 |
| 退出登录 | /api/auth/logout | POST | 注销当前 refreshToken（Redis 中删除） | 无需 |

### 10.2 详细接口

#### 10.2.1 刷新 Token

- **URL**: `/api/auth/refresh`
- **方法**: `POST`
- **认证**: 无需认证（使用 refreshToken 本身验证身份）
- **请求体**: 
  ```json
  {
    "refreshToken": "JWT refresh token字符串"
  }
  ```
- **响应**: 
  ```json
  {
    "code": 200,
    "message": "刷新成功",
    "data": {
      "accessToken": "新的access token（有效期30分钟）",
      "refreshToken": "新的refresh token（旧token自动吊销）"
    }
  }
  ```
  > 每次刷新会轮换（rotate）refresh token，旧的 refresh token 会被吊销

#### 10.2.2 退出登录

- **URL**: `/api/auth/logout`
- **方法**: `POST`
- **认证**: 无需认证
- **请求体**: 
  ```json
  {
    "refreshToken": "待注销的JWT refresh token字符串"
  }
  ```
- **响应**: 
  ```json
  {
    "code": 200,
    "message": "已退出登录",
    "data": "已退出登录"
  }
  ```
