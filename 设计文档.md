GET
获取用户的书架列表
userId (路径)

GET
获取特定小说的书架记录
userId, novelId (路径)

GET
检查是否已收藏
userId, novelId (查询参数)
/api/bookshelf/user/{userId}/count
GET
统计收藏数量
userId (路径)

POST
加入书架
{userId, novelId} (JSON)
/api/bookshelf/remove
DELETE
移出书架
{userId, novelId} (JSON)
/api/bookshelf/remove/batch
DELETE
批量移出书架
{userId, novelIds[]} (JSON)
/api/bookshelf/reading-progress
PUT
更新阅读进度
{userId, novelId, chapterId} (JSON)
使用场景
用户收藏小说: 点击"加入书架"按钮
查看我的书架: 展示用户收藏的所有小说
继续阅读: 记录上次阅读的章节，方便续读
取消收藏: 从书架移出不喜欢的小说
批量管理: 一次性清理多本小说
特色设计
唯一索引: (user_id, novel_id) 确保用户不会重复收藏同一本小说
自动同步: 收藏/取消收藏时自动更新小说的 collectCount
阅读进度: 记录最后阅读的章节 ID 和时间
排序优化: 按更新时间倒序排列，最近阅读的在前面
批量操作: 支持批量移出书架
